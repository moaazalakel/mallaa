# منصة الملاءة - Mallaa Platform

منصة إلكترونية متكاملة ومهنية لقياس مستوى نضج وجودة خدمات التشخيص التربوي والنفسي المقدمة للطلبة ذوي الإعاقة في سلطنة عمان.

## المميزات

- **لوحة تحكم للأخصائيين**: إدارة الحالات، التقييمات، والأنشطة على مستوى المحافظة
- **لوحة تحكم للمشرفين**: متابعة شاملة لجميع المحافظات، التدقيق، والتقارير الاستراتيجية
- **نظام إشعارات**: متابعة الإجراءات المعلقة
- **تقارير تفاعلية**: رسوم بيانية متقدمة باستخدام Recharts
- **تصدير PDF**: حفظ لوحات التحكم كملفات PDF
- **واجهة عربية RTL**: تصميم متجاوب وحديث

## التقنيات المستخدمة

- **React 19** - مكتبة واجهة المستخدم
- **Vite 7** - أداة البناء
- **Tailwind CSS 4** - إطار عمل CSS
- **React Router v7** - التوجيه
- **Recharts** - الرسوم البيانية
- **date-fns** - معالجة التواريخ
- **localStorage** - تخزين البيانات (نظام تجريبي)

## التثبيت والتشغيل

### المتطلبات

- **Node.js** 18 أو أحدث
- **npm** 9 أو أحدث (أو yarn/pnpm)

### خطوات التثبيت

1. **استنساخ المستودع**:
```bash
git clone <repository-url>
cd mallaa
```

2. **تثبيت الحزم**:
```bash
npm install
```

3. **تشغيل المشروع في وضع التطوير**:
```bash
npm run dev
```

4. **فتح المتصفح**:
افتح المتصفح على `http://localhost:5173`

### بناء المشروع للإنتاج

```bash
npm run build
```

الملفات المبنية ستكون في مجلد `dist`

### معاينة البناء

```bash
npm run preview
```

## النشر على GitHub Pages

### إعداد المشروع

1. **تأكد من إعداد `base` في `vite.config.js`**:
```javascript
base: '/mallaa/', // يجب أن يطابق اسم المستودع
```

2. **تأكد من إعداد `basename` في `src/main.jsx`**:
```javascript
<BrowserRouter basename="/mallaa">
```

3. **تثبيت `gh-pages`**:
```bash
npm install --save-dev gh-pages
```

4. **إضافة سكريبت النشر في `package.json`**:
```json
{
  "scripts": {
    "deploy": "npm run build && gh-pages -d dist"
  }
}
```

### النشر

```bash
npm run deploy
```

سيتم نشر الملفات على `https://<username>.github.io/mallaa/`

### ملاحظات مهمة للنشر

- تأكد من تفعيل GitHub Pages في إعدادات المستودع
- اختر فرع `gh-pages` كمصدر للنشر
- قد يستغرق النشر بضع دقائق حتى يظهر التحديث

## ملاحظات مهمة

- هذا نظام **تجريبي** - البيانات محفوظة في localStorage فقط
- لا يوجد نظام مصادقة حقيقي - للحسابات التجريبية فقط
- المرفقات غير مدعومة في النسخة التجريبية
- البيانات تُعاد توليدها تلقائياً عند تغيير الإصدار في `seedData.js`

## التطوير

### تشغيل Linter

```bash
npm run lint
```

### هيكل البيانات

البيانات التجريبية تُولد تلقائياً في `src/data/seedData.js` وتُحفظ في localStorage. عند تغيير `currentVersion` في الملف، سيتم إعادة توليد البيانات.
